## Problem
The @workspace/api-server service (REPO, RAILPACK builder) fails at the BUILD_IMAGE stage. Railpack runs `pnpm install --frozen-lockfile --prefer-offline`, which exits with code 1 and the following error:
```
ERR_PNPM_LOCKFILE_CONFIG_MISMATCH  Cannot proceed with the frozen installation. The current "overrides" configuration doesn't match the value found in the lockfile
Update your lockfile using "pnpm install --no-frozen-lockfile"
```
The start command is `pnpm --filter @workspace/api-server start` (set via dashboard). No railway.json or railway.toml config file is present.

## Diagnosis
The pnpm-workspace.yaml file contains an extensive `overrides` section that excludes platform-specific binaries for esbuild, rollup, lightningcss, and tailwindcss oxide (e.g., `esbuild>@esbuild/linux-arm64: '-'`, `rollup>@rollup/rollup-darwin-arm64: '-'`, etc.). These overrides were modified after pnpm-lock.yaml was last generated, so the lockfile no longer reflects the current workspace configuration. When Railpack runs `pnpm install --frozen-lockfile`, pnpm detects the discrepancy and refuses to proceed.

The triggering commit (4438853) added `exclude-links-from-lockfile=false` to .npmrc to try to fix the error, but that setting controls a different pnpm behavior and does not resolve the overrides mismatch. The same error occurred in the previous deployment (b4579f03), confirming the lockfile has been stale across multiple deployments. No subsequent commits have been pushed to main, so the issue remains unresolved on HEAD.

## Fix
1. Locally, in the repository root, run:
   ```
   pnpm install --no-frozen-lockfile
   ```
   This regenerates pnpm-lock.yaml to match the current overrides in pnpm-workspace.yaml.
2. Commit the updated pnpm-lock.yaml file and push to main.
3. This ensures that when Railpack runs `pnpm install --frozen-lockfile`, the lockfile exactly matches the workspace configuration and the build succeeds.

This fix cannot be performed by a code agent — it requires executing the pnpm CLI to regenerate the lockfile.
